KEY FINDINGS
================================================================================

ARCHITECTURE PATTERN:
- Distributed budgeting (no single BudgetManager)
- Each extension negotiates its own context allocation
- Local safety margins (80% for search, 200 lines for memory, etc.)
- Global hard caps (2000 lines, 50KB output)
- Cascading truncation and temp file overflow

MODEL CONTEXT WINDOWS:
- Claude Opus 4.6:   1,000,000 tokens (128k output)
- Claude Sonnet 4.6: 1,000,000 tokens (64k output)  
- Claude Haiku 4.5:  200,000 tokens (64k output)

SAFETY MARGINS:
- Search budgeting: 80% of maxTokens (20% reserve)
- Memory injection: 200 lines + 25KB hard caps
- Output truncation: 2000 lines + 50KB hard caps
- Context7 docs: Configurable (default 5k, max 10k tokens)

CACHING STRATEGIES:
- Search results: 10-minute LRU (50 entries max)
- Context7 docs: Session-long (cleared on shutdown)
- RTK gains: 15-second memoization (5s spawn timeout)

CONTEXT INJECTION HOOKS:
- before_agent_start: Memory loading and system prompt injection
- session_compact: Process state recovery after context reset
- turn_end: Auto-dream consolidation checks
- session_shutdown: Auto-extract memory consolidation
